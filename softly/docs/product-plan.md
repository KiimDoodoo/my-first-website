## 3. 신규 기획: 다이어리 꾸미기 ("소프틀리 꾸미기")

### 3.1 배경과 목표

- 한국 기록 앱 사용자에게 꾸미기(다꾸)는 기록을 "해야 하는 일"에서
  "하고 싶은 놀이"로 바꾸는 핵심 장치 (레모리·하루콩 등의 인기 요인)
- 목표: **기록의 정서적 소유감**을 높여 자발적 재방문을 만든다.
  꾸미기는 보상이 아니라 표현이어야 한다 — 잘함/못함이 없어야 한다.

### 3.2 설계 원칙 (Softly 다움)

1. **차분한 꾸미기**: 모든 스티커·테마는 뮤트 톤 팔레트 안에서만.
   채도 높은 빨강, 네온, 반짝임·깜빡임 효과 금지 (광과민 안전 규칙 유지)
2. **압박 없는 잠금 해제**: 연속 기록(스트릭) 보상 금지.
   누적 기록일 기준(빠진 날 무관)으로만 천천히 열린다.
   잠금 항목은 회색 실루엣으로만 표시 — "놓쳤다"는 느낌을 주지 않는다
3. **꾸미지 않아도 완전한 기록**: 꾸미기는 전부 선택. 안 해도 어떤
   화면에서도 빈자리가 보이지 않는다
4. **과금 없음** (MVP 범위): 모든 꾸미기 요소는 무료

### 3.3 기능 상세

#### A. 스티커 (Phase 1)
- 체크인 완료 화면과 마음 서랍 카드에서 **오늘 기록에 스티커 1개**를
  붙일 수 있다 (탭 한 번, 교체·제거 가능)
- 스티커는 SVG 8~12종 시작: 새싹 🌱, 조약돌, 초승달, 구름, 찻잔,
  단풍잎, 물결, 작은 별 (전부 세이지·웜 뉴트럴 톤)
- 표시 위치: 마음 서랍 카드 모서리, 오늘 기록 상단, 새싹 달력의
  해당 날짜(스티커가 새싹을 대체)

#### B. 테마 (Phase 1)
- 설정에서 앱 전체 색 분위기 선택: 세이지(기본) / 웜 베이지 /
  안개 라벤더 / 깊은 숲(다크 강조)
- 모든 테마는 기존 명도 대비 기준과 다크 모드를 유지

#### C. 캐릭터 꾸미기 (Phase 2)
- 헬퍼 캐릭터에 작은 액세서리: 잎모자, 목도리, 동그란 안경, 베레모 등
- 누적 기록일로 해제 (예: 3일 → 잎모자, 7일 → 목도리, 14일 → 안경)
  - 카피 예시: "함께한 날이 7일이 되었어요. 목도리가 어울리겠어요."
  - 금지 카피: "연속 7일 달성!" / "스트릭 유지!"
- 홈·체크인·완료 화면의 캐릭터에 공통 적용

#### D. 마음 서랍 꾸미기 (Phase 2)
- 한 줄 카드의 종이 색(4종 뮤트 톤) 선택
- 월 달력 위 작은 제목 손글씨 폰트 옵션 (시스템 폰트 + 1종)

### 3.4 데이터 모델 확장

```ts
interface DailyCheckIn {
  // ...기존 필드
  sticker?: string;          // 스티커 id (예: "sprout", "moon")
}

interface UserProfile {
  // ...기존 필드
  theme?: "sage" | "warm" | "lavender" | "forest";
  characterItems?: string[]; // 착용 중인 액세서리 id
}
```

- 누적 기록일은 저장하지 않고 체크인 배열 길이로 계산 (파생값)
- 스티커·테마는 JSON 내보내기/가져오기에 자동 포함

### 3.5 구현 단계

| 단계 | 내용 | 규모 |
| --- | --- | --- |
| Phase 1 | 스티커 8종 + 테마 4종 | 화면 3곳 수정, 1~2일 |
| Phase 2 | 캐릭터 액세서리 + 서랍 꾸미기 | 캐릭터 컴포넌트 확장, 2~3일 |

### 3.6 스토어 반영

- 앱스토어 설명에 추가할 불릿:
  "• 다이어리 꾸미기 — 차분한 스티커와 테마로 나만의 기록 공간 만들기"
- 라이프스타일 카테고리 적합성을 오히려 강화 (다이어리·꾸미기는
  라이프스타일의 대표 장르)
